Upgrade Routine v1.1 => v1.2 ONLY:

Make sure you have a structural and schematic backup before you begin.

Please read these instructions carefully. Upgrading is entirely at your own risk:

Upgrade Instructions:

v1.2 is an upgrade for search engine friendly urls. If your server doesn`t support htaccess this will not be necessary.

To upgrade to v1.2, follow the instructions below. Structure has been maintained in this version.

1: Access your database via your server control panel and run the following update file. Note that if you changed the prefix, you`ll need to update this file before you run it.

/docs/upgrade_data/v1_2/upgrade.sql

2: Open your current language file in your text editor and update/add the variables in the following file:

/docs/upgrade_data/v1_2/new_lang.txt

3: Replace the following files in your 'admin' folder:

/admin/data_files/home.php (Updated)
/admin/data_files/settings.php (Updated)
/admin/index.php (Updated)
/admin/images/logo.gif (Updated) (Optional)

4: Replace or add the following files in your installation url. The 'inc/defined.inc.php' is a new file and sees the defined vars moved from the index.php file, to make future updates easier. These will need resetting. So, for example, if you had the captcha disabled, you`ll need to reset this var in the new file.

classes/class_mail.inc.php (Updated)
inc/header.inc.php (Updated)
inc/functions.php (Updated)
inc/defined.inc.php (New)
index.php (Updated)

If you updated the mail class file for HTML e-mails, you`ll need to open that file in your text editor and set the $html var to true. This is an easier option in v1.2.

5: Replace the following templates. If you made extensive edits to these files, you can update each file manually. All you need to change are the urls. The new vars are all prefixed '_URL' and you`ll also need to add the base href tag in the header.tpl.php file. If you made no edits, just replace the files.

templates/popups/contact.tpl.php (Updated)
templates/popups/friend.tpl.php (Updated)
templates/header.tpl.php (Updated)
templates/recipe.tpl.php (Updated)

6: Finally, add the .htaccess file into your recipe root folder. If you already have an .htaccess file in place, update to include recipe directives. Note that if your server doesn`t support this you will see a server error. If that happens, remove it:

.htaccess (New)

7: Log in to your administration area and update the url path to include the full url to your recipe installation. No trailing slash. Also, enable mod rewrite if you want search engine friendly urls.

8: If required, change url text variables in the 'inc/defined.inc.php' file. Read the instructions carefully before changing these variables. Also, make sure other vars are reset as necessary.

Upgrade complete. Test to make sure everything is working ok.
